Resting Places: architecture and data flow

DirectoryOS build, cemeteries vertical. Static Astro 7 site, restingplaces.org. Every stage reports real output before the next runs.

1. Meta-list fan-out crawl (reusable module)

Seed: Wikipedia Lists_of_cemeteries. BFS over list pages to closure, depth cap 3: index pages fan out to country lists, country lists to state and city lists.

120 sub-lists crawled, 13,018 raw entity links via MediaWiki generator=links + pageprops QIDs.

↓

2. Entity confirmation + dedupe

Links confirmed as cemeteries via Wikidata SPARQL P31/P279* Q39614, never guessed from titles. 10,859 unique QIDs, 2,443 confirmed, 8,416 rejected (cities, people, wars).

Output: data/seed-Claude.csv

↓

3. Enrichment, licensed sources only

Wikidata batches: coordinates, founding date, admin region, area, heritage id, official site, Commons image name, Find a Grave id (link-out only). Nominatim fallback geocoding, accepted only when OSM classes the hit as a burial ground. Google Places skipped: no key, so hours are omitted sitewide by the freshness gate.

Raw responses cached to data/cache/ as provenance evidence.

↓

4. Interment knowledge graph (reusable module, the core asset)

Reverse property query: persons with P119 = cemetery, requiring an enwiki article + death date in the query itself. Private individuals are structurally unreachable.

75,372 edges, 70,270 persons, 1,678 cemeteries with notable interments. CC0. Ships whole at /api/interments.json.

↓
BLOCKING GATE

5. Image rights

Commons imageinfo + extmetadata per candidate. PASS only PD / CC0 / CC BY / CC BY-SA (share-alike flagged). 1,887 pass, 19 fail, failures ship text-forward. No genealogy-site photos, no generated imagery.

↓

6. Classify + data model

Fixed vocabularies over sourced facts: type from P31 labels + heritage, era from founding date. Unmatched stays untagged and is counted. Zod schema validates every record at build; a null is a null, never a guess.

↓

The cemetery-to-person knowledge graph

Generalizes to any entity-to-entity relation: birthplaces (P19), headquarters (P159), filming locations (P915). The module is the pairing of a meta-list crawl for the entity set with a reverse-property SPARQL sweep for the relation.
